# 阶段路线图

这份文档描述产品阶段。更细的工程执行顺序见 [开发路线图](development-roadmap.md)。

## Phase 1：文档设计

当前阶段。

目标：

- 明确 fox 总体愿景。
- 明确健身先行。
- 明确训练主持闭环。
- 明确 AI 与状态机边界。
- 明确下一阶段数据库和本地 Web Runtime UI 所需概念。

完成标志：

- 文档结构完整。
- JSON schema 可校验。
- 下一阶段可以直接设计数据库表和本地 Web Runtime UI 原型。

## Phase 2：数据库设计

目标：

- 把文档中的健身概念落成数据库草案。
- 设计 Entry 与 WorkoutSession 的关系。
- 明确事件日志模型。

范围：

- 不做 iPad。
- 不做 ASR/TTS。
- 不做姿态识别。

## Phase 3：本地 Web Runtime UI 原型

目标：

- 在本地浏览器完成一次训练。
- 支持计划、确认、动作执行、休息倒计时、反馈、调整、总结。
- 支持节奏动画、可视化倒计时、触控友好的反馈按钮。

最小界面：

- 今日状态 check-in
- 计划确认
- 主训练执行
- 组后反馈
- 休息倒计时
- 总结确认

## Phase 4：iPad App

目标：

- 把本地 Web Runtime UI 验证过的流程搬到 iPad。
- 优先大按钮、倒计时、当前动作、总结确认。

不做：

- 复杂 3D 形象。
- 自动姿态识别。
- 全自动语音控制。

## Phase 5：ASR/TTS

目标：

- 用语音降低训练中操作成本。
- ASR 输出结构化反馈事件。
- TTS 播放教练消息。

原则：

- 不复用旧实现。
- ASR/TTS 只做输入输出层。

## Phase 6：传感器与姿态识别

目标：

- 研究加速度计计数。
- 研究摄像头姿态估计。
- 对少数动作做辅助验证。

原则：

- 不影响手动反馈主流程。
- 准确率不足时只作为参考。
