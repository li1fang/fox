# 项目愿景

fox 的长期愿景是成为一个私人的生活操作系统。

它不追求一开始就全自动，而是先可靠地记录用户生活中的关键事件，再逐步把记录变成提醒、复盘、建议和陪伴。

## 四个领域

fox 长期覆盖四个领域：

- 饮食：吃了什么，大概多少热量，什么时候吃。
- 医疗：买了什么药，医生说了什么，处方如何执行。
- 健身：今天练什么，练到哪里，强度是否合适。
- 记账：花了多少钱，花在什么类别。

## 统一底座

四个领域共享同一个底座：`Entry`。

每条 Entry 都描述一件发生过的事：

- 时间
- 类型
- 原始证据
- AI 提取草稿
- 用户确认结果
- 后续复盘价值

## 健身为什么先行

健身是第一条主线，因为它已经有过真实原型经验。

用户曾经做过一个 TUI 训练服务，它没有主动感知，也不漂亮，但具备可用内核：倒计时、反馈、调整、总结。fox 的第一版应该复刻并强化这个内核，而不是被视觉形象、自动计数或复杂模型拖走。

## 长期形态

长期 fox 可以拥有：

- iPad 上的训练和生活记录界面。
- 后台数据库。
- 本地 Linux 4090 算力中心。
- ASR/TTS 语音输入输出。
- 饮食照片识别。
- 医疗 OCR 和处方整理。
- 有限的健身传感器或姿态识别。

但这些都必须建立在稳定的记录系统和健身主持闭环之上。
